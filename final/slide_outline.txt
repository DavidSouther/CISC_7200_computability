=== II. THEOREMS (14 slides) ===

4. [bullets] Setup
   - S, a set of partial computable functions
   - I_S, its index set
   - Trivial: S is N or S is empty
   - Non-trivial: non-empty partitions

> Let's do a bit of setup. We start with S, a set of partial computable functions. Partial computable means that we're sure some natural numbers as inputs to this function will end with a value coming out, but we're not sure which ones.

> We have an index set I_S of that set S, so we can assign a number to each function in S. We're not talking about the concrete case of the properties of any single number in any single function, but rather a class of functions as a black box. If S is "functions that calculate whether an input number is even", any particular function might be testable with 0, 1, 2, 5, and 8 and we'd be OK with it. But we want to say, of _any_ function: is it a function that figures out if a given value is even?

> So S is the kinds of functions we want to identify, I_S is a way to check if some function is in S. A couple points for completeness: S is "trivial" if it's empty (no functions have this property, like "the function is an elephant") or if it's every function. Every function is a function, ok, yes, that's true thank you.

5. [bullets] Rice's theorem, formal
   - If S is non-trivial, then I_S is undecidable
   - No general decision procedure for "is this function in S"

> So, Rice's theorem: if S is non-trivial, then I_S is undecidable. For the proof:

6. [diagram] Rice diagram
   - The set of partial computable functions, split into "with property" and "without property"
   - Caption: undecidable which side a given index lands on

> Picture the world of partial computable functions cleaved into two sides. Which side an index falls on is the thing we can't decide.

7. [bullets] Rice's theorem, proof
    - Pick $a, b \in \mathbb{N}$ with $a \in S$ and $b \notin S$
    - Define $f(e, x)$ by cases:
      - $f(e, x) = \varphi_b(x)$ when $e \in I_S$
      - $f(e, x) = \varphi_a(x)$ when $e \notin I_S$
    - By s-m-n, there is $e$ with $\varphi_e(x) = f(e, x)$
    - If $e \in I_S$ then $\varphi_e = \varphi_b$, but $b \notin S$
    - If $e \notin I_S$ then $\varphi_e = \varphi_a$, but $a \in S$
    - Either way: contradiction

> a and b are two naturals, where a is an index of a function in S and b is an index of a function not in S. We're going to construct f using the same procedure that we did in Turing's proof of halting, and in every category theory diagram. f with e as the first argument is a total computable function defined by taking the opposite of the assumptions in the first slide.

> With an application of s-m-n, we get from f to phi, and then directly to the contradiction. If phi_e is phi_b, e is both in and not in I_S. And vice versa.

8. [diagram] Rice's theorem, in category theory form
   - Left middle: S
   - Right middle: decidable
   - Top left: I_S
   - Top right: f
   - Arrow: Left to top left, I_S is index set of S
   - Arrow: Top left to top right, cases
   - Arrow: Left middle down along bottom to right middle: direct
   - Arrow: Top right to right middle: the contradiction

> And here it is again, drawn as a category-theory diagram. Same proof, different lens.

9. [bullets] Rice's theorem, contrapositive
   - If I_S is decidable, then S is trivial
   - The only decidable function properties are no property at all

> Turn it around to the contrapositive. Assume the index set is, in fact, decidable. Then what do we know about S? The only thing we can know is that S is either empty, or everything. This on its own isn't interesting, but hold the thought.

10. [bullets] Rice-Shapiro, formal
    - Rice: if S is non-trivial, then I_S is undecidable
    - Shapiro: f in S iff there exists finite g, a subset of f, with g in S

> The word that should jump out here right away is "finite". So we had Rice, which says I_S is undecidable. Let's keep with the same set and index set, and think through it a bit different. This says that some function f is in S if and only if another function, this time a finite function g, is in the index set. So we said the index set itself is undecidable, but Shapiro is saying hey, you can start to tell whether f is in S by checking just a finite chunk of f.

11. [diagram] Rice-Shapiro diagram
    - The Rice picture, plus several points on one side and one on the other, straddling the boundary

> Here it is as a picture. We've got the partitioning of S here, and we have some candidate function f. These points are a subset of f, and in this case, g suggests that f is a reasonable candidate.

12. [diagram] Rice-Shapiro diagram with refutation
    - The Rice picture, plus several points on one side and one on the other

> Uh oh, here's another point in g, and this one is on the wrong side! So this finite g says hey, f is definitely not in the Index Set, since the index set can only generate things on the left side!

13. [bullets] Rice-Shapiro, proof sketch
    - Assume I_S r.e., f in S, no finite g subset of f is in S
    - For each n, define h_n(x): output f(x) only while $\varphi_n(n)$ has not yet halted
    - $\varphi_n(n)$ never halts $\Rightarrow$ h_n = f, so h_n in S
    - $\varphi_n(n)$ halts at stage s $\Rightarrow$ h_n is a finite restriction of f, so h_n not in S
    - n not in K iff index of h_n in I_S
    - I_S r.e. would make complement of K r.e., contradicting Turing

> Sketch by reduction from the halting problem. Assume the index set is r.e. and Shapiro's witness rule fails for some f. We build, for each natural n, a partial computable h_n that mimics f as long as $\varphi_n(n)$ hasn't halted yet. If $\varphi_n(n)$ never halts, h_n is just f and lands in S. If $\varphi_n(n)$ halts at some stage, h_n is a finite restriction of f, which by assumption is not in S.

> So n is not in K precisely when h_n's index is in I_S. An r.e. index set therefore makes the complement of the halting problem r.e., which Turing rules out. So Shapiro's witness rule has to hold.

14. [diagram] Rice-Shapiro, in category theory form
    - Top left: $\overline{K}$ (complement of halting)
    - Top right: $h_n$, parametric construction
    - Bottom left: I_S, assumed r.e.
    - Bottom right: r.e.
    - Arrow: top left to top right, the construction n maps to h_n
    - Arrow: top right to bottom left, membership decided by I_S
    - Arrow: bottom left to bottom right, the assumption
    - Arrow: top left to bottom right, the impossible reduction

> Same proof, same shape. The complement of K reduces to I_S through the parametric h_n. Assuming I_S is r.e. would give K-complement r.e., closing the square at a contradiction.

15. [bullets] Rice-Shapiro, contrapositive
    - If a function f in S has some finite g subset of f with g not in S, then I_S is not r.e.

> And this is the much easier way to say it, as the contrapositive.

16. [bullets] Plain English, all four
    - Rice: non-trivial behavioral properties are undecidable
    - Rice contrapositive: decidable means trivial
    - Rice-Shapiro: behavior is fixed by finite evidence
    - Rice-Shapiro contrapositive: one finite counterexample, and the index set is incorrect

17. [bullets] Bridge to history
    - Rice-Shapiro: one counterexample is enough
    - Mathematicians arrived here in 1956
    - Along with the rest of the scientific world

> This really feels super obvious, and intuitive? It does for me at least. But going through history, accepting this is anything but obvious. It's actually pretty damned hard, for a lot of folks.

> Let's talk some history.

18. [bullets] Section title

--- Archimedes ---

19. [bullets] Archimedes, portrait
    - Image: Domenico Fetti, "Archimedes Thoughtful" (1620), or a classical bust
    - Caption: Archimedes of Syracuse, c. 287 to 212 BCE

    - Syracuse, Magna Graecia
    - Mathematician, engineer, astronomer
    - Killed at the siege of Syracuse by a Roman soldier, mid-equation

> So this is Archimedes, one of those Greek figures we hear so much about. He's a bit towards the end of the Hellenistic world; Aristotle and Alexander the Great were both dead 50 years before Archimedes was born. So he's working from all the writings of the Pythagoreans, and he's got all of Euclid in his back pocket, but he doesn't have things like Algebra or the concept of "zero" as a number. If you asked him, he'd tell you the natural numbers start at 1. Still, he very nearly invented calculus, so he's a pretty smart dude.

20. [bullets] Archimedes, the question
    - Hiero II has a beautiful crown
    - The artisan says it's pure gold
    - Can't damage or destroy the crown

> On the falsification experiment, he's actually here as a judge rather than a questioner. His king, Hiero the second, has just gotten this gorgeous crown, and he needs to know if the artisan is being honest. How do you figure out if something is pure gold?

> If you just weigh it, you don't know if it's pure gold, or gold-silver alloy of the same weight. You could melt it and try separating out any silver, but that kinda defeats the purpose of having this crown! It's already made, so we can't, like, have a guard look over their shoulder the entire time, either.

> The story goes that this was the first time Archimedes had come across this problem any way (we technically don't know if someone figured it out before and didn't write it down). So Archimedes is sitting around thinking about this problem, and it's said that he fills his bath tub all the way up to the brim. (It's the ancients, not the medieval, so they still took baths.) He gets in, and a bunch of water spills out... how much water? Well, the same volume as he takes up!

> He jumps out screaming "Eureka!" and runs naked down the street with an experiment in hand, a technique to ask a falsifiable question (or refutable question). He tells his idea to the king (still naked), they do the experiment, and execute the dishonest artisan. So the king got his crown _and_ didn't pay for it.

21. [diagram] The displacement experiment
    - Fill the vessel to the spout
    - Submerge a weight of gold equal to the crown's, catch the overflow in A
    - Refill, submerge the crown, catch the overflow in B
    - Compare A and B
    - Caption: if the crown displaces more water than gold of equal weight, the claim is refuted

> Here's a quick look at the experimental apparatus. The interesting thing here is that we're using this indirect measurement to find "the real thing". This is an early computation, using a theory to derive an expected value, then measuring against that value. Keep this in mind.

22. [bullets] Archimedes, takeaway
    - The artisan made a claim
    - An indirect measurement can refute the claim

23. [diagram] Archimedes, in category theory form
    - Same trapezoid shape as the Rice diagram
    - Caption: claim, indirect measurement, refutation

--- Bridge: Galileo ---

24. [diagram] Galileo, two pictures
    - Image left: sunspot drawings from "Letters on Sunspots" (1613)
    - Image right: two bowling balls dropped from a tower
    - Caption: predictions specific enough to be wrong

> We're jumping ahead a ways. Anyone recognize this guy? Galileo was a lens maker. Lenses were super important for Venetians. As sea-faring traders, it was really useful to know sooner than later whether that ship on the horizon was your own traders coming back with silks and spices, or if it was your enemy showing up with guns to take your silks and spices. But as an early Enlightenment scientist, he's also pointing them up at the sky and everywhere else he can.

> He's got two things that are salient to us here. Both are falsifications. The first was simply looking at things, in this case, the sun. Catholic doctrine stated the heavenly bodies were perfect and unmoving. Galileo, with his telescope, pointed it at the sun and was intrigued to find a blemish, and one that moved! The sun wasn't perfect, and it wasn't fixed. After being tried for blasphemy and ostensibly renouncing his claims, (and, this is probably apocryphal), he says "Eppur si muove", "And yet, it moves".

> The second takes aim at Aristotle, who says that heavy things fall faster. Galileo claims a cannon ball and a marble dropped at the same time land at the same time. Actually that was an experiment in Delft, by de Groot and Stevin. Galileo does a thought experiment that arrives at an intriguing contradiction, but that will have to wait for another time.

--- Newton ---

25. [bullets] Newton, portrait
    - Image: Godfrey Kneller's 1689 portrait
    - Caption: Isaac Newton, 1643 to 1727
    - Trinity College, Cambridge, 1665
    - Principia Mathematica, 1687
    - President of the Royal Society

> Moving ahead the year after Galileo's death, Isaac Newton is born in England. He takes refuge from the plague in 1665, sees the apocryphal apple drop from a tree (or fall on his head), and begins wondering about the ideas that we now call "force". If "force" can reach out over a distance, what does that mean? He spends the rest of the summer inventing calculus, proving the inverse square law of gravitational attraction, predicts exactly Kepler's planetary motion from first principles and the return of Halley's comet, and then like the total jackass he is doesn't publish it for another 22 years.

26. [bullets] Newton, the claim
    - Universal gravitation, inverse square
    - Predicts planetary orbits, tides, falling apples
    - One theory for the heavens and the earth

> Using the simple principles of the three laws of motion (Inertia, Force as Mass by Acceleration, and the principle of action and reaction) and also inventing all of calculus, Newton perfectly predicted _where_ planets would be. To this point, Kepler had shown that planets did move in an ellipse, and with laborious effort could calculate which ellipse. But Newton turned that on its head and showed that the ellipse was a consequence of a simpler principle, and with the simpler principle, the labor of calculating a trajectory was much improved.

27. [diagram] Halley's comet
    - Image: contemporary engraving of the 1759 return
    - Caption: Halley predicts 1758, the comet returns 1759

> So Edmund Halley is a royal astronomer (again, if you can see a ship further away, you can better prepare to fight them). And he's got ideas on when a certain comet will re-appear. To this point, the orbits of the planets were well established, but, like, why would a comet follow that? It's a comet, not a planet... but ok, why wouldn't it?

> So Halley tries to calculate it. And he gets it... kinda right? He's off by a few months. Which given their equipment is pretty darned good! He ends up visiting Newton, asking Newton to check his notes, and Newton's like "your calculations are wrong" and Halley's like "how do you know that?" (Halley was doing everything with trigonometry, which... SUCKS... for orbital mechanics) and Newton's like "Because I already figured it out 20 years ago"

> Halley's like "bullshit" and Newton goes to his shelf and pulls out his notes from Cambridge (the ones that, you know, invented calculus) and sure enough there's his calculations for Halley's Comet with a slightly more accurate prediction. Newton called his shot and nailed it. Halley's impressed, and actually pays himself to get Newton's Principia published.

> Pretty good, yeah?

28. [diagram] Mercury's orbit
    - Image: precession of the perihelion, exaggerated
    - Caption: Newton predicts X, observation gives X plus 43 arcseconds per century, the theory misses

> Well..... it's not quite right. Mercury has an additional orbital motion that Newton can't explain.

29. [bullets] Newton, takeaway
    - Newton calls a shot, then nails it
    - A theory survives by surviving the next observation
    - Mercury is the called shot the theory missed
    - Einstein opens a new regime, Newton becomes a subset

> So what does Newton give us in the world of falsifiability? He's still calculating things proactively. He happens to be getting it right enough of the time to be valuable. Which brings us to the guy who finally figured out what was actually going on.

--- Popper ---

30. [diagram] Karl Popper
    - Image: black and white portrait of Popper at the LSE
    - Caption: Karl Popper, 1902 to 1994
    - Vienna, schooled at the edge of the Vienna Circle
    - Emigrates to New Zealand before WW2
    - London School of Economics
    - "The Logic of Scientific Discovery," 1934

> Karl Popper is one of the more influential philosophers of science in the 20th century. He grew up in Vienna, and his parents were close friends of Sigmund Freud's sister, Rosa Graf. He was also very sympathetic to early communist movements, but we'll see in a bit he sours on Marxism. He's able to flee Vienna before Nazi rule, to New Zealand, but after publishing his first seminal work "The Logic of Scientific Discovery."

31. [bullets] Popper, the foil
    - Freud explains every behavior, in any direction
    - A theory that forbids nothing predicts nothing
    - The asymmetry: confirmation is cheap, refutation is decisive

> The first part of Logic of Discovery is a critique of Freud and Marx. Again, he grows up around Freud, and notices this pattern. Whenever Freud makes a claim, whatever a patient says or does just becomes more evidence for the theory. There's no behavior that would prove him wrong.

> This is even more problematic when Popper views Marxism. While the ideas of Marx and Engels in Das Kapital seem very persuasive, the violence of the Russian Revolution and the rise of Nazism really sour him on their scientific validity. Even moreso than Freud, Marxist revolutions in practice require continuous ad-hoc refinement for any new data point that comes up. They just keep changing the so-called theory to meet whatever the leadership wants, that it's not what anyone could consider "scientific". This is later carried forward in "The Open Society", which, aside, is a shockingly relevant text for US politics today.

32. [bullets] Popper, the rule
    - A claim is scientific if it forbids something
    - Science calls its shot before it takes it
    - One missed shot is enough

> Popper's claim is that a scientific theory is one that can be falsified by evidence. A pseudoscientific claim may sound plausible, and a pseudoscientific practitioner will see examples _confirming_ their theory nearly anywhere they look. But that's easy and wrong. We must look for examples that _disprove_ our claims to understand when we've found where a theory fails.

33. [diagram] Yanofsky-style cone
    - Left: the countable world (objects, observations)
    - Right: the cone of functions, algorithms, programs
    - Caption: mappings between the world and the language we use to talk about it

> Let's look at this through a familiar lens. The right should look familiar, it's the same tower of functions, algorithms, and programs we've seen throughout the course. We have all possible functions on the bottom. Then there's all algorithims, of which a countably infinite number satisfy a certain function. And then there's actual programs, again a countably infinite number of programs for each specific algorithm.

> But on the left, we have something that came up a bit more recently, and that's the notion of "the world" or "reality". I call it "the things one could count". The left is anything that "actually exists", that you could go up to and interact with. The right is in each of our minds - it's conceptual, theoretical, language and stories. It's kind of been hiding, both through this course and throughout Western thought.

34. [bullets] Three eras, one discipline
    - Archimedes shows something that could go either way
    - Newton calculates exactly what to expect in a range of circumstances
    - Popper describes how to narrow a theory to make it stronger
    - Goedel, Turing, Rice, Shapiro: the same, mathematically

> So this is a very brief overview of three figures in history looking at that first question - how do you know and trust a claim is right? If the person making the claim leaves room for it to be falsified, you can have a pretty good idea that they're honest. At least, they can be comfortable putting a bounds on what they're saying.

=== IV. PRACTICE: DITHERING AND CONSTRAINT (8 slides) ===

35. [bullets] Section title
    - Practice: Dithering and Constraint
    - Two boundaries on an undecidable middle
    - Inner: tests. Outer: types.

> OK and in this last part, we're going to switch from a math talk and a history talk to a software engineering talk. How do we ask these kinds of question in software? Rice' theorem says that mathematically, given a function, we can't really do much interesting things with that function, we can't look at it and get anything useful out. The only thing we can do with it, well, give it an input, wait around for a while, and... see what happens?

> But that's not very interesting. Instead, we're going to attack it from two directions. From the bottom, we're going to apply rice-shapiro and a technique I'll call dithering to find where the boundary of the function's properties are, and on the other we're going to attack down from the program side, since we can say interesting things about programs.

36. [diagram] World and function, business example
    - The Yanofsky-style cone, redrawn
    - Label a business process on the world side (e.g. "is this transaction fraudulent")
    - Label the matching function on the program side
    - Caption: the engineer's job lives between these two arrows

> Here's our cone again, but we've got a lot more detail. Over on the left, in the world, we have a bunch of transactions that people made. Over on the right, we need a function that classifies a transaction as is_fraud and not is_fraud.

37. [bullets] Dithering and Rice-Shapiro
    - The Rice-Shapiro picture, applied to a working function
    - Pick a point that should land inside, run it, observe
    - Pick a point that should land outside, run it, observe

> We're going to start with a unit test. This point here? This one should definitely be labeled fraud. And here's a point, this one is not fraud. Ok, that's nice. But we've still got this boundary we want to find.

38. [diagram] Loose dithering
    - Many points scattered around the boundary
    - Some land inside (witnesses), some outside (refutations)
    - Caption: each point is a called shot

> And what we'll do is just keep "dithering", that is, in an organized manner we'll just try a whole bunch of points around here. And as we keep going, we'll find exactly where the line in the function is. If we have enough points, we can be confident that all of them lie within the partition that we're interested in, the fraud side.

39. [bullets] Dithering as unit testing
    - A unit test is a called shot at behavior
    - Landed: a witness, the function is inside the set, for this input
    - Missed: a refutation, the function is not what we claimed
    - The suite raises the lower bound on the index set, witness by witness

> This is unit testing, or property testing, or dithering around the area we're interested in. We looked at the real world, we created a theory of what constitutes a fraudulent transaction, and we found a function that's a proof of that theory. So long as every point keeps getting classified correctly, we have confidence in our claim that this function is the truth of the question.

> A couple notes for completeness - the function or the program in this case is actually doing both F and not F at the same time, so it's giving us the total partition. And the assertion is "this point lands on the correct side of the partition", which while one step removed from the formalism of Rice' theorem, I hope you're OK following me there.

> Second, this does gloss over that this function is bounded, so there's technically a third case where the function fails to find an answer within a certain number of steps.

40. [bullets] Type systems
    - Types call shots at composition, not behavior
    - The compiler is the referee
    - A type error is a refutation before the shot is even taken

> I do want to go out on a bit of a limb and come at this from the other direction, because again I think this is really important to understand at least my distinction between the mathematician, the computer scientist, and the software engineer. Mathematicians have said these nice wonderful theorems that tell me I can't just use a black box to understand if I have the right function. That's great, it means I have a job and I get paid a lot of money to find things at the top of the diagram that map down to the bottom.

> But when programming, I'm not typing down here in the function space, the function never "exists" anywhere. It's on the right hand side, in language. I'm actually working up here, in the real world in text, which maps to the top right in syntactical programs set. And if I'm in programs, there are some very strong proofs and theorems we can apply to the lists and trees that create programs!

41. [diagram] Type system as outer fence
    - The full space of programs
    - A type system carves out a permitted region
    - Caption: ill-typed programs are excluded before runtime

> The compiler is a total computable function that looks at the finite set of typable programs, and makes provable assertions about their behavior. Programs are finite, so this is cool. But what that does is also gives us constraints going down the stack! If I can prove that this particular program halts, syntactically (it has no loops, or the loop can be shown to complete), then I know that the function it represents down here has certain useful properties, like the function f that is encoded by P will halt. That's neat!

42. [diagram] Two bounds on the index set
    - The Rice-Shapiro picture, with both bounds drawn
    - Lower bound (witnesses from tests) creeping up from inside
    - Upper bound (refusals from types) creeping down from outside
    - The undecidable middle, shaded, between them
    - Caption: the gap between the bounds is where the bugs live

> Zooming in on the partition down here, we see that we have the area the boundary, and we're dithering around it with unit tests. But now that I have a specific program, that I claim represents this function, we also have an outer bounds of what this program could achieve. And that means we have a much smaller area to dither around! The work we need to do to show that there is no counter example is much smaller, and much more tractable in practice
